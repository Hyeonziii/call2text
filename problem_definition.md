# 문제 정의서 — Call2Text

> 기억해조 5조 | 신한은행 AI 해커톤 | 작성일: 2026-06-27

---

## 1. 해결할 문제

### 배경

신한은행 콜센터 상담원은 하루 평균 수십 건의 고객 상담을 진행한다. 통화 종료 후 상담원은 상담 내용을 수동으로 요약해 안내 문자를 작성·발송한다. 이 과정에서 다음과 같은 비효율이 발생한다.

| 문제 | 현황 |
|------|------|
| 수동 문자 작성 시간 | 건당 평균 3~5분 |
| 개인정보 노출 위험 | 수동 작성 시 고객 정보 실수로 포함 가능 |
| 상품 안내 일관성 부재 | 상담원마다 추천 상품·링크가 다름 |
| 상담 품질 데이터 미활용 | 과거 상담 사례가 신규 상담에 반영되지 않음 |

### 핵심 문제 정의

> **"금융 상담 직후, 상담 내용 기반 맞춤형 고객 안내 문자를 자동 생성하여 상담원의 사후 처리 시간을 단축하고 고객 경험을 일관되게 향상시킬 수 있는가?"**

---

## 2. 가설

| # | 가설 | 검증 방법 |
|---|------|-----------|
| H1 | 정규식 기반 PII 마스킹으로 주요 개인정보(전화번호·계좌번호·주민번호 등)를 95% 이상 제거할 수 있다 | 마스킹 전·후 텍스트 비교, 패턴별 적중률 측정 |
| H2 | TF-IDF char n-gram 검색으로 금융 도메인 유사 상담을 cosine similarity 0.1 이상으로 검색할 수 있다 | 상담 유형별 샘플 쿼리 10개 × top-4 결과 평가 |
| H3 | GPT-4o-mini는 상담 유형 분류에서 키워드 기반 fallback 대비 높은 정확도를 보인다 | 50개 샘플에 대해 두 방법 정확도 비교 |
| H4 | RAG 컨텍스트를 포함한 LLM 프롬프트가 컨텍스트 없는 프롬프트보다 더 적절한 SMS 초안을 생성한다 | 블라인드 A/B 평가 (상담원 5명, 문자 20쌍) |

## 3. 성공 기준

### 기술적 지표

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| SMS 초안 생성 성공률 | ≥ 95% | OpenAI API 호출 성공 여부 |
| 생성 문자 글자수 준수율 (≤ 300자) | ≥ 95% | 생성된 draft 길이 분포 |
| 상담 유형 분류 정확도 | ≥ 85% | 레이블 있는 25번 데이터 기준 |
| RAG 검색 평균 유사도 | ≥ 0.08 | 상담 유형별 쿼리 top-1 score |
| PII 마스킹 적중률 | ≥ 95% | 전화번호·계좌번호 패턴 단위 |

### 사용자 경험 지표

| 지표 | 목표 |
|------|------|
| 문자 초안 생성 소요 시간 | ≤ 5초 (API 응답 포함) |
| 상담원 수정 없이 그대로 발송 비율 | ≥ 50% (정성 평가) |
| 상담원 만족도 | 5점 척도 ≥ 4.0점 |

## 4. 데이터의 한계

### 4.1 데이터 가용성 제약

| 데이터셋 | 제약 사항 |
|---------|-----------|
| **25번 금융상담** (AI Hub) | `.zip.part0` 형식으로 전체 데이터의 일부만 포함. 완전한 zip 파일이 아니므로 대용량 상담 이력 접근 불가 |
| **36번 CoT 추천** (AI Hub) | 상품 추천 근거(CoT)는 있으나 실제 가입 여부·결과 데이터 없음 → 추천 성과 검증 불가 |
| **소비자 특성 데이터** | 연령·성별 등 인구통계 필드만 존재, 상담 내용 텍스트와 직접 연결되지 않음 |

### 4.2 모델링 한계

| 한계 | 상세 |
|------|------|
| **TF-IDF 의미 검색 불가** | char n-gram 기반으로 단어 형태 유사성만 반영. 의미적으로 같은 표현("대출 한도" vs "대출 가능 금액")을 다르게 처리 |
| **실시간 금융 정보 부재** | 실제 금리·한도 등 최신 상품 정보가 포함되지 않아 생성 문자에 부정확한 수치가 포함될 수 있음 |
| **LLM 환각(Hallucination)** | GPT-4o-mini가 존재하지 않는 상품명이나 조건을 생성할 가능성 존재 → 상담원 검토 필수 |
| **언어 편향** | 학습 데이터가 표준어 위주로 구성되어 사투리·구어체 상담에서 STT 정확도 및 분류 정확도 저하 가능 |

### 4.3 운영 한계

| 한계 | 상세 |
|------|------|
| **RAG 인덱스 디스크 캐시 없음** | 앱 재시작마다 TF-IDF 인덱스를 재빌드 (데이터 규모에 따라 30~120초 소요) |
| **단일 파일 아키텍처** | 모든 로직이 `app.py`에 집중되어 기능 확장 및 단위 테스트 어려움 |
| **개인정보 처리 방침** | 마스킹 후에도 LLM API(OpenAI) 서버로 상담 내용이 전송됨 → 실운영 시 데이터 처리 계약 필요 |
